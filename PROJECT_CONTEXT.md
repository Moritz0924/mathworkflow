# 项目上下文

## 定位

本项目帮助参赛者把 ChatGPT 网页端的建模能力与 Codex 的本地工程能力组合成可复现、可审计的数学建模工作区。它不是自动答题 API，也不是训练平台。

## 当前版本

- 版本：v4 thin formal workflow
- 工作方式：一场比赛一个工作区
- AI：ChatGPT 网页端 + Codex
- 交接：本地结构化文件
- 状态：8 阶段顺序状态机 + 5 个人工闸门
- 事实来源：数据、已运行代码、冻结结果和合同

## 核心目录

```text
00_problem/             题面和附件清单
01_task_analysis/       问题拆解
02_literature/          可选文献证据
03_data/                原始/处理数据、字典与质量报告
04_eda/ 04_eda_code/    EDA 报告与可复现代码
05_model/               模型、假设、公式与备选路线
06_code/                唯一正式求解代码目录
07_results/             真实输出与冻结报告
08_figures/             图表与证据设计
09_paper/               论文草稿
10_ai_logs/handoffs/    ChatGPT/Codex 不可变交接
11_review/              审稿、修订与校验报告
12_submission/          唯一最终提交目录
13_prior_db/            可选历史经验资料，不参与状态机
14_contracts/           事实与证据合同总线
mmwf/                   v4 工作流内核
prompts/formal_v4/      v4 结果合同提示词
```

## 决策边界

- ChatGPT 负责“应该采用什么模型、如何解释和写作”。
- Codex 负责“本地事实是否成立、代码能否复现、数字是否一致”。
- Codex 发现实现问题可以修复；发现建模或结论冲突必须阻塞并退回 ChatGPT。
- 控制器只做协议和状态治理；人类决定所有关键冻结与最终提交。
