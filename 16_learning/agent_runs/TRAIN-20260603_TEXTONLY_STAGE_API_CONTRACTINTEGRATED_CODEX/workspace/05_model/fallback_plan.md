# Fallback Plan – 模型降级与失败恢复

| 子问题 | 主模型 | 失败模式 | 降级备选 | 降级触发条件 | 恢复后验证 |
|---|---|---|---|---|---|
| Q1 | AHP+熵权 TOPSIS | 判断矩阵不一致（CR>0.1）或指标共线性VIF>10 | PCA+等权或专家直接打分 | AHP一致性检验失败或熵权变异系数异常 | 排序稳定性再检验 |
| Q2 | LightGBM/RF | 测试集性能显著低于验证集（过拟合）或关键特征缺失 | ElasticNet或简单时间序列基线（如 Holt-Winters） | 交叉验证集与测试集误差差距>20% | 保留测试集重新评估 |
| Q3 | 多目标规划 | 非线性太强或求解时间过长 | 遗传算法/粒子群优化 | 非线性项占比>30%或CPLEX/Gurobi无法在1小时内收敛 | 启发式结果多次运行取稳健解 |

## 全局降级策略
- 若数据质量不满足任意主模型要求，优先使用非参数方法（如秩和比法评价、KNN预测、贪婪式优化）作为最低基线。
- 任何降级必须记录在案，并在论文中说明理由与影响。
