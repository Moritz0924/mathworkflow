# 训练增强点

## 系统层面
- **TEP001**: 合同验证脚本（validate_contracts.py）应包含主题一致性检查，自动检测合同中是否出现与锁定主题无关的术语。
- **TEP004**: 修订合同更新过程，确保修复操作不会引入新的事实或漂移，可能需要引入变更跟踪机制。

## 提示词层面
- **TEP002**: 各阶段提示词应植入主题锁定语句，禁止模型生成与问题无关的内容，尤其是AQI等历史项目术语。
- **TEP005**: 审稿器权限描述需添加明确的白名单，防止权限滥用或跨越到论文正文。

## 闸门层面
- **TEP003**: 模拟人工闸门需联机检查脚本，在关键阶段自动运行主题漂移检测，并提供阻塞能力。
- **TEP006**: 最终导出闸门应硬性要求所有合同校验、审稿评分达标，否则拒绝通过。

这些点反映了当前训练运行中暴露的流程缺陷，建议纳入后续工作流版本升级。
