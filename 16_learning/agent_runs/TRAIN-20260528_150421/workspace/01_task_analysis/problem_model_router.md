# 题型与模型族权重路由

## 目标

先判断题目本质，再决定正文写作和图片生成的资源分配。不要把所有建模题都写成同一个论文壳。

## 最小判断单元

每一问必须填写：

1. `problem_type`：题目类型。
2. `model_family`：主模型族。
3. `data_shape`：数据形态。
4. `output_type`：最终输出。
5. `risk_focus`：最容易被评委扣分的位置。

## problem_type 可选值

| 类型 | 适用场景 | 写作重心 |
|---|---|---|
| 数据评价型 | 多指标打分、排序、分级、综合评价 | 指标体系、权重、评价合理性、解释性 |
| 预测回归型 | 预测数值、趋势、概率、类别 | 数据处理、变量关系、检验、误差控制 |
| 优化决策型 | 路径、调度、资源分配、组合方案 | 目标函数、约束、算法、方案比较 |
| 机理仿真型 | 传播、演化、动力系统、状态转移 | 假设、机理、参数、仿真验证 |
| 机器学习型 | 分类、识别、推荐、复杂非线性预测 | 特征、训练、对比、泛化验证 |
| 综合开放型 | 多阶段、多模型串联、政策建议 | 任务拆解、模型链路、结果整合 |

## model_family 可选值

`优化决策`、`统计评价`、`预测回归`、`机理仿真`、`机器学习`、`空间网络`、`综合集成`。

## 路由规则

1. 若题目要求“最优、安排、调度、路径、分配、选址”，优先进入 `优化决策`。
2. 若题目要求“评价、排序、等级、综合指数”，优先进入 `统计评价`。
3. 若题目要求“预测、拟合、影响因素、趋势”，优先进入 `预测回归`。
4. 若题目出现“传播、演化、动力学、模拟、状态变化”，优先进入 `机理仿真`。
5. 若数据为图像、文本、高维特征或明确要求识别分类，优先进入 `机器学习`。
6. 若同一问同时满足多个规则，按“最终交付物”定主模型族，其他模型作为辅助模型。

## 输出要求

完成 `problem_model_profile.csv` 后运行：

```bash
python scripts/route_weight_config.py
```

生成：

- `09_paper/active_weight_config.csv`
- `08_figures/active_figure_plan.csv`
- `08_figures/active_visual_profile.md`
