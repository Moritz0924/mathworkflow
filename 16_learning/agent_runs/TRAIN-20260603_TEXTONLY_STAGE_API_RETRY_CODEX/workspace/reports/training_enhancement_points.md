# 训练增强点

## 系统层面
- 阶段锁定机制应更加严格，当前 paper_full 可在前期阶段未完成时被调用，导致合同缺失。建议增加前置条件校验 (SYS-01)。
- paper_full 启动时应自动扫描并报告缺失的上游合同 (SYS-02)。

## 提示词层面
- 增加合同缺失处理的具体指令，明确如何记录缺口而不阻塞全文组装 (PRM-01)。
- 要求代理输出显式缺件清单，而非在日志中静默忽略 (PRM-02)。

## 闸门层面
- 模拟闸门的 formal_effect 字段应与正式闸门严格保持一致，避免验证误报 (GAT-01)。
- 在人工闸门中包含对未解决缺口列表的逐项确认 (GAT-02)。
