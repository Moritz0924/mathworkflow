# 训练增强点

## 系统层面

1. **合同校验强化** (TEP-001)：`validate_contracts.py` 应检测 `figure_contract.csv` 中同一行使用分号分隔多个 `result_id` 的情况，要求每条记录仅包含单个 `result_id`，确保链接关系清晰可追踪。当前已通过人工重写解决，若脚本前置此规则可避免同类问题。
2. **图表视觉验证** (TEP-002)：沙盒环境中缺乏图表渲染反馈，建议在未来 sandbox 中集成 MCP 视觉技能，至少对中文文本渲染效果进行截图比较，以确保导出的 SVG/PDF 满足可读性要求。

## 提示词层面

3. **导出阶段 PPT 边界** (TEP-003)：`final_export` 阶段提示词注明 PPT 为可选，但未具体说明生成内容必须完全来源于已批准论文，不得新增数字、图表或结论。建议细化该约束，并给出正面示例和禁止示例。
4. **中文可见文字硬性检查** (TEP-004)：当前提示词要求“所有最终可见文字为中文”，但缺乏可执行的检查步骤。建议在 `final_export` 中加入自动化扫描步骤（例如提取所有图片中的文本元素并确认无英文），并将结果写入检查报告。

## 闸门层面

5. **闸门中的中文合规确认** (TEP-005)：模拟人工闸门日志目前仅确认流水，未专项核对中文合规。建议在闸门问题清单中加入“最终导出物是否已确认所有可见文字为中文（必要缩写除外）？”一项，并记录确认结果。
