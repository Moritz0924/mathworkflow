# 图表视觉规范 v2.4

## 全局原则

1. 图表必须服务结论，不做装饰性堆砌。
2. 单张图必须有 `main_message`，图注必须解释结论，而不是复述标题。
3. 变量颜色在全文保持一致。
4. 正文图优先导出 SVG/PDF，PNG 不低于 300 dpi。
5. 默认绘图样式不得裸用，禁止默认配色。
6. 图中文字必须使用中文，且通过渲染检查确认无乱码。
7. 同一论文至少包含 4 个图表家族，任一图类占比不超过 35%。

## 字体与字号

| 元素 | 建议 | 下限 |
|---|---:|---:|
| 标题 | 12-14 pt | 11 pt |
| 坐标轴标题 | 10-11 pt | 9 pt |
| 坐标轴刻度 | 9-10 pt | 8 pt |
| 图例 | 9-10 pt | 8 pt |
| 注释 | 8-9 pt | 8 pt |
| 图中节点文字 | 9-11 pt | 8 pt |

## 中文字体规则

Python 图表必须执行：

```python
plt.rcParams["font.sans-serif"] = ["Noto Sans CJK SC", "Microsoft YaHei", "SimHei", "PingFang SC", "Arial Unicode MS"]
plt.rcParams["axes.unicode_minus"] = False
```

导出后必须打开 SVG/PDF/PNG 检查：标题、图例、坐标轴、注释、负号、单位无乱码。

## 颜色规则

- 每篇论文主色板最多 2 套。
- 每张图主色不超过 5 个。
- 红色只用于风险、异常、下降、警告。
- 连续变量优先使用单调或发散色板，不使用彩虹色。
- 分类变量优先使用低饱和高级色，避免刺眼荧光色。

## 线条与网格

- 主线宽度：2.0-2.5。
- 辅助线宽度：0.8-1.2。
- 网格线使用浅灰，仅辅助读数。
- 关键阈值线必须标注来源或含义。

## 审美门禁

| 维度 | 进入正文最低要求 |
|---|---|
| 图文契合 | 图直接支持相邻段落结论 |
| 信息密度 | 至少包含 2 层信息：趋势+阈值、结果+比较、结构+权重等 |
| 颜色 | 非默认配色，符合变量语义 |
| 中文 | 无乱码，术语统一 |
| 多样性 | 不与前后图重复同类表达 |
| 导出 | SVG/PDF 或 300 dpi PNG |

## 禁止项

- 不使用彩虹色作为分类色。
- 不在同一图中使用过多主色。
- 不生成无单位、无坐标轴、无图例的数据图。
- 不让图例遮挡核心信息。
- 不把代码截图、软件截图当成正式图。
- 不让柱状图和折线图垄断正文。
