# 训练增强建议报告

## 系统层面 (System)
- **阶段锁定机制局限**：当前设计下，主题漂移修复需在较早阶段完成，若锁定则无法后续矫正。建议允许在特定闸门（如auto_review）后由人工触发解锁关键阶段。（高优先级）
- **合同校验反馈**：沙盒训练中缺少实时合同校验，导致fail项积压到最终阶段。应在每个阶段结束时自动运行检查并记录。（高优先级）
- **写入权限**：final_export当前无权写入08_figures，无法紧急生成缺失的图形文件，可能需要扩展允许路径或引入回退机制。（低优先级）

## 提示词层面 (Prompt)
- **导出与修复边界**：final_export提示词明确禁止新增论文事实，但训练中常遇到前期缺陷需修复。应在提示词中说明沙盒模拟下允许的有限修复范围。（中优先级）
- **闸门模拟指令**：沙盒覆盖规则虽定义了模拟闸门，但各阶段实际调用时缺少明确操作步骤，导致日志记录不一致。建议在沙盒提示词中包含样板。（中优先级）

## 闸门层面 (Gate)
- **豁免追踪**：当前模拟闸门依赖手动豁免记录，缺少自动化。未来可设计豁免数据库，关联到具体fail项，便于审计。（中优先级）
- **日志格式**：simulated_human_gate_log.csv设计合理，但需要确保各阶段均记录且formal_effect明确。（低优先级）
