# 模型降级与备选计划

## Q1 综合评价
- **方案 A**：若熵权-TOPSIS 结果受数据分布影响，改用 CRITIC 权重法或等权法。
- **方案 B**：若需考虑定性因素，切换至模糊综合评价。
- **方案 C**：若区域间差异不明显，采用 PCA 降维后聚类。
- **触发条件**：敏感性分析中排名相关性 < 0.8，或人工认为权重不合理。

## Q2 因素识别
- **方案 A**：若 XGBoost 过拟合，尝试随机森林（降低树深度）或 Lasso 回归（强正则化）。
- **方案 B**：若特征重要性波动大，改用弹性网络（Elastic Net）稳定选择。
- **方案 C**：若存在空间相关性，扩展为空间误差模型（SEM）或地理加权回归（GWR）。
- **触发条件**：交叉验证 R² < 0.6，或 SHAP 解释与专业知识严重冲突。

## Q3 资源优化
- **方案 A**：若 ILP 求解时间过长或不可行，改用遗传算法（GA），以初始可行解作为种子。
- **方案 B**：若效果系数存在显著不确定性，改用鲁棒优化或目标规划。
- **方案 C**：若需多目标权衡，采用多目标遗传算法（NSGA-II）生成 Pareto 前沿。
- **触发条件**：求解器返回无可行解，或人工认为线性假设过强。

## 通用降级原则
- 优先保证结果的可解释性和复现性。
- 当复杂模型与简单模型表现相近时，论文中同时报告，以简单模型作为稳健性对照。
- 所有降级变更须记录到 `14_contracts/revision_tasks.csv` 并在阶段总结中说明。
