# 训练增强点报告

本报告记录了在`final_export`阶段观察到的系统、提示和闸门方面的可改进点，供后续工作流优化参考。

## 系统层面
1. **文件读取受限**：在沙盒中代理无法读取现有文件，导致合同修复、内容同步等操作必须基于假设，增加了引入不一致的风险。建议在沙盒中至少提供合同和关键草稿的只读访问。
2. **合同只读限制与遗留错误**：final_export阶段合同只读，但上游可能未彻底解决合同缺陷（如figure link缺失），需建立阶段入口合同自检，失败则拒绝进入最终导出。

## 提示层面
1. **阶段提示完整性**：当前final_export提示未充分说明遇到合同不一致或缺失时应采取的行动，建议明确“合同不符合则暂停导出并指示上游修复”。
2. **与open fail队列集成**：训练沙盒中open fail项目未自动注入stage prompt，代理需人工解析，易遗漏。建议将活跃fail项作为prompt的一部分显式传递。

## 闸门层面
1. **闸门日志标准化**：模拟闸门日志格式在多次迭代中可能不一致，建议提供标准CSV模板并强制列存在。
2. **自动合同校验闸门**：应在final_export阶段伪执行`check_gates.py`和`validate_contracts.py`（即使沙盒中无法真实运行），并记录预期结果，作为闸门条件。

这些增强点若被采纳，可提升工作流在受限环境下的鲁棒性和可复现性。
